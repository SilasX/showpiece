The Problem

Whenever you write a web page you invent new and inconsistent markup that is hard to maintain.

Suppose you want to add a main menu. It's not always an easy task. You may need submenus, command groupings, icons, tooltips, selected state, disabled state, etc.

You need simple semantic markup that gives you room to grow. One markup to rule them all.

Simple menu, no style

Let's start simple. Just a plain menu with some items.

Oh, and you can edit the definition. Try adding some items. Inspect the resulting markup with your developer tools.

[ { "url": "/", "text": "Home" }, { "url": "/contact", "text": "Contact" } ]

Groups

Many menus split their items into groups.

Also notice how marking an item disabled changes the generated markup.

[ { "text": "File", "groups": [ { "items": [ { "url": "/open", "text": "Open" }, { "url": "/save", "text": "Save As...", "disabled": true } ] }, { "items": [ { "url": "/print", "text": "Print" } ] } ] } ]

Nested submenus

Let's start simple. Just a plain menu with some items.

Oh, and you can edit the definition. Try adding some items.

[ { "url": "/mammals", "text": "Mammals", "items": [ { "url": "/mammals/puppies", "text": "Puppies" }, { "url": "/mammals/kitties", "text": "Kitties" } ] }, { "url": "/reptiles", "text": "Reptiles", "items": [ { "url": "/reptiles/alligators", "text": "Alligators" } ] }, { "url": "/other", "text": "Mystery" } ]

Item Descriptions

You can describe each item and group in a menu. This can be used to clarify the item, or to show changing status.

[ { "url": "/", "text": "Home", "description": "a nice place" }, { "url": "/contact", "text": "Contact" } ]

Demo: Twitter

Showpiece generates clean, semantic and flexible markup. It can be styled however you like.

For example, here's the Twitter menu. Look how the logical structure gets transformed

[ { "text": "", "id": "gear", "description": "Settings and help", "groups": [ { "items": [ { "id": "profile", "url": "#", "text": "John Doe", "description": "Edit profile" } ] }, { "items": [ { "url": "#", "text": "Direct Messages", "selected": true }, { "url": "#", "text": "Lists" } ] }, { "items": [ { "url": "#", "text": "Help" }, { "url": "#", "text": "Keyboard Shortcuts" }, { "url": "#", "text": "Twitter Ads" } ] }, { "items": [ { "url": "#", "text": "Settings" }, { "url": "#", "text": "Sign out" } ] } ] } ]

Demo: Facebook

An interesting example where the menu item text appears in a tooltip while the description is always visible.

The CSS uses the "data-nav" property to apply icons to each menu item.

[ { "url": "#", "id": "news", "text": "News Feed", "selected": true }, { "url": "#", "id": "messages", "text": "Messages", "description": "9" }, { "url": "#", "id": "friends", "text": "Find Friends", "description": "6" }, { "url": "#", "id": "events", "text": "Events" }, { "url": "#", "id": "photos", "text": "Photos" } ]
